Question 1a: What type of authentication are we using here (currently)? Does it use any keys?

Answer: We are using basic username/password authentication. This is shown through the process of connecting to InfluxDB where the command requires -username and -password. As it uses username and password to authenticate, key is not used in authentication. 

Question 1b: Both TLS (encryption) and crypto authentication use public-private key pairs.
For TLS encryption what keys are used when the client sends a message to the server?
For crypto authentication, explain how the server can verify a message is from a given
client?

Answer: In TLS encryption, the client uses the server's public key to encrypt the data while the server uses its private key to decrypt the data. In crypto authentication, the client generates a signature using its private key, which then the server verifies the signature using its client's public key.

Question 2: Here we created a pair of asymmetric keys. What are their names? Which one
is the public key and which one is the private key?

Answer: The two asymmetric keys used for this pair is server-cert.pem and server-key.pem. server-cert.pem is the public key and server-key.pem is the private key.

Question 3: What is a certificate authority (CA) for public keys? What kind of attack can
a CA prevent?

Answer: Certificate authority is a trusted entity that issues and signs digital certificates. A CA can prevent attacks that are done in between client and the server, such as server impersonation. They make a thorough verification to make sure clients connect to the real servers.
