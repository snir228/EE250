Question 1a: What type of authentication are we using here (currently)? Does it use any keys?

Answer: We are using basic username/password authentication. This is shown through the process of connecting to InfluxDB, where the command requires -username and -password. Since it uses a username and password for authentication, the key is not used. 

Question 1b: Both TLS (encryption) and crypto authentication use public-private key pairs.
For TLS encryption what keys are used when the client sends a message to the server?
For crypto authentication, explain how the server can verify a message is from a given
client?

Answer: In TLS encryption, the client uses the server's public key to create a symmetric key used by both client and server to encrypt and decrypt data. In crypto authentication, the client generates a signature using its private key, which the server then verifies using the client's public key.

Question 2: Here we created a pair of asymmetric keys. What are their names? Which one
is the public key and which one is the private key?

Answer: The two asymmetric keys used for this pair are server-cert.pem and server-key.pem. server-cert.pem contains the public key and the server key.pem contains the private key.

Question 3: What is a certificate authority (CA) for public keys? What kind of attack can
a CA prevent?

Answer: Certificate Authority is a trusted entity that issues and signs digital certificates. A CA can prevent attacks that occur between the client and the server, such as server impersonation. They conduct thorough verification to ensure clients connect to the real servers.
